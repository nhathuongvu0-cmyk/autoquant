# autoquant v2.0

量化策略自動優化器 — **嚴格防止過擬合版本**

## 核心原則

**只有 Test 集盈利的策略才算成功！**

```
Train (2018-2022) → 用於開發
Val (2023-2024)   → 用於調參
Test (2025+)      → 最終驗證
```

## 通過標準

| 指標 | Train | Val | Test | 說明 |
|------|-------|-----|------|------|
| PF | >1.0 | >1.2 | **>1.2** | Test 必須盈利 |
| DSR | >2 | >1.5 | **>1.0** | 統計顯著 |
| 穩定性 | - | - | |val-test| < 30% | 不能大幅衰減 |

## 實驗循環

```
1. 修改 strategy.py
2. git commit
3. python3 backtest.py > run.log 2>&1
4. 檢查 Test 結果
5. 如果 test_pf > 1.2 且 test_dsr > 1.0 → 保留
6. 否則 → 丟棄
7. 重複
```

## 過擬合警告

如果看到這種結果：
```
Val:  PF=1.77, DSR=6.74 ✅
Test: PF=0.86, DSR=-999 ❌
```

**立即丟棄！** 這是典型的過擬合。

## 可探索方向

1. **更簡單的策略** — 參數越少越好
2. **不同市場環境** — 牛/熊市分開測試
3. **更長的回看期** — 避免短期噪音
4. **不同資產** — 考慮 ETH 等

## 禁止行為

- ❌ 只看 Val 結果就保留
- ❌ 調參超過 10 個變量
- ❌ 忽略 Test 集虧損
- ❌ 多次重跑選最好的
